# Spotify Landscape Player 仕様書

## 1. プロジェクト概要
Spotifyの再生状況をリッチに表示し、操作を可能にするWebアプリケーション。
横画面（Landscape）での利用を前提としたデザインを採用し、PWA（Progressive Web App）としてネイティブアプリのような体験を提供する。
GitHub Pagesを利用してホスティングを行う。

## 2. コア機能
### ビューワー機能
- **Now Playing**: 現在再生中の曲のジャケット画像を大きく表示
- **Track Info**: 曲名、アーティスト名を表示
- **Up Next**: 次に再生予定の曲リスト（Queue）を3~5曲程度表示

### コントロール機能
- **Playback Control**: 再生 / 一時停止
- **Navigation**: 次の曲へスキップ
- **(Optional) Previous**: 前の曲へ戻る

### PWA / システム機能
- **Installable**: ホーム画面に追加可能
- **Standalone**: ブラウザのアドレスバー等を隠し、全画面アプリとして動作
- **Authentication**: Spotify アカウントによるログイン（OAuth 2.0 PKCEフロー）

## 3. 技術スタック案
- **Frontend Framework**: React (TypeScript)
- **Build Tool**: Vite
- **Styling**: TailwindCSS
- **State Management**: React Context or lightweight store (e.g. Zustand)
- **PWA**: vite-plugin-pwa
- **Hosting**: GitHub Pages
- **Deployment**: GitHub Actions

## 4. UI/UX コンセプト
- **Landscape First**: 横向きレイアウトに最適化。左側にジャケット、右側に情報とリスト、などの2カラム構成を検討。
- **Visuals**: ジャケット画像から抽出した色を用いた動的な背景グラデーションや、Glassmorphism（すりガラス効果）を取り入れたプレミアムなデザイン。
- **Touch Friendly**: タッチ操作しやすい大きめのボタン配置。

## 5. 認証フローについて
GitHub Pagesは静的ホスティングであるため、バックエンドサーバーを必要としない **Authorization Code Flow with PKCE** を採用する。これにより、クライアントサイドのみで安全にSpotify APIのトークンを取得・更新できる。
