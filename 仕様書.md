# Spotify Landscape Player 仕様書

## 1. プロジェクト概要
Spotify の再生状況をリッチに表示し、操作を可能にする鑑賞特化型の Web アプリケーション。
横画面（Landscape）での利用を前提としたレイアウトを採用し、PWA としてネイティブアプリのような体験を提供する。
GitHub Pages を利用してホスティングを行う。

## 2. コア機能
### ビューワー機能
- **Now Playing**: 再生中の曲のジャケット画像を大きく表示。タップでフルスクリーン切り替え。
- **Lyrics (同期歌詞)**: `lrclib.net` API を活用した歌詞表示。再生位置に合わせた自動ハイライトとスムーズなスクロール。
- **Up Next**: 次に再生予定の曲リスト（Queue）を表示。
- **Track Info**: 曲名、アーティスト名を表示。

### コントロール機能
- **Playback Control**: 再生 / 一時停止。
- **Gesture Control**: 画面の左右スワイプによる「次の曲」「前の曲」へのナビゲーション。
- **Navigation**: ボタン操作によるスキップ機能。

### PWA / システム連携機能
- **Wake Lock API**: 再生中（フルスクリーン時）に画面がスリープするのを防止。
- **Browser Notifications**: 曲が切り替わった際に、バックグラウンドでも通知を表示（PWA Service Worker 連携）。
- **Orientation Lock**: モバイル端末での横向き表示を試行。
- **Authentication**: Spotify アカウントによるログイン（OAuth 2.0 PKCE フロー）。

## 3. 技術スタック
- **Frontend**: React 19 / TypeScript / Vite 7
- **Styling**: Tailwind CSS / Framer Motion (アニメーション・ジェスチャー)
- **Icons**: Heroicons
- **API**: Spotify Web API / lrclib.net (歌詞データ)
- **State Management**: Custom Hooks (ポーリング & 時間補間ロジックによる滑らかな同期)
- **PWA**: `vite-plugin-pwa`
- **Hosting/CI/CD**: GitHub Pages / GitHub Actions

## 4. UI/UX コンセプト
- **Landscape First**: 2カラム構成（左：ジャケット・操作、右：キュー・歌詞のタブ切り替え）を採用。
- **Smooth Interaction**: Spotify API のポーリング（3秒）を補間する 100ms 周期のタイマーにより、歌詞の移動や再生バーを滑らかに表現。
- **Premium Design**: ダークテーマを基調とし、Glassmorphism や動的なスケーリングアニメーションを取り入れたモダンなデザイン。

## 5. 認証フロー
サーバーレス環境（GitHub Pages）に対応するため、**Authorization Code Flow with PKCE** を採用。トークンの自動リフレッシュ（有効期限の5分前実行）および 401 エラー時の自動リトライにより、安定した通信を確保。
